## Reconciliation loop

Every 60s (existing reconciler interval) the phase registry runs:

- `catchall` (domain-scoped): pushes `catchall_target` â†’ `x:Domain.catchAllAddress`.
- `disclaimer` (domain-scoped): pushes `disclaimer_enabled + disclaimer_text` â†’
  `x:SieveSystemScript` named `jabali-disclaimer-<domain>`.
- `autoresponder` (mailbox-scoped): pushes `email_autoresponders` row â†’
  `VacationResponse/set`.
- `forwarders` (mailbox-scoped): pushes all enabled rows for the mailbox â†’
  alias list on `x:UserAccount.aliases` + concatenated sieve script
  `jabali-fwds`.
- `mailbox_share` (mailbox-scoped): pushes `mailbox_shares` rows owned by the
  mailbox â†’ `Mailbox.shareWith` patch on the mailbox's INBOX.

Operator-side edits via Stalwart admin console are considered drift and
overwritten on the next tick.

## Troubleshooting

- **Reconciler logs** â€” `journalctl -u jabali-panel | grep -iE 'phase|autoresponder|catchall|disclaimer|forwarder|share'` shows every tick's
  per-phase run. Errors include the feature name and domain/mailbox id.
- **Stalwart JMAP timeout** â€” raise via `panel-api` config; phases use 10-15s
  per-call timeout.
- **Empty Logs tab on a user with no domains** â€” expected; UI paginates empty.
- **"mailbox not yet registered"** from autoresponder/forwarder agent â€” the
  mailbox account hasn't authed to Stalwart yet (ADR-0045 lazy registry).
  The reconciler retries next tick; first successful auth populates the
  registry.
- **Disclaimer not applied** â€” Spike A unverified on the host; check that
  Stalwart supports the sieve extensions listed in the script's `require`
  line (`envelope`, `body`, `replace`, `foreverypart`). Run
  `stalwart-cli get x:SieveSystemInterpreter` to list enabled capabilities.
